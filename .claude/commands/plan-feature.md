# /plan-feature

Prima di scrivere qualsiasi codice, produci un piano strutturato per la feature richiesta.

Il piano deve includere:
1. **Obiettivo** — cosa deve fare questa feature in una frase
2. **File coinvolti** — lista esatta dei file da creare o modificare (con percorso completo)
3. **Sequenza di implementazione** — step numerati, ogni step corrisponde a un'unità atomica di lavoro
4. **Dipendenze** — nuovi package npm necessari, se presenti
5. **Rischi** — cosa potrebbe andare storto, con soluzione proposta
6. **Test da eseguire** — come verificare che la feature funzioni

NON scrivere codice. NON iniziare l'implementazione.
Attendi conferma esplicita dell'utente prima di procedere.
